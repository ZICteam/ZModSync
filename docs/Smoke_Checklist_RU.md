# ModSync Smoke Checklist

Текущая версия мода: `1.0.6`

Этот чеклист нужен для быстрой ручной проверки после важных изменений перед выпуском новой версии.

## Когда использовать

- после изменения сетевой логики
- после изменения manifest generation
- после изменения download flow или cleanup
- после изменения UI синхронизации
- перед публикацией новой версии мода

## Базовая проверка

- мод собирается без ошибок через `gradlew build`
- новая версия указана в `gradle.properties`
- новая версия добавлена в `CHANGELOG.md`
- документация обновлена, если изменение влияет на поведение мода

## Проверка подключения

- сервер запускается с ModSync без ошибок в логе
- клиент видит сервер в списке
- статус сервера определяется без зависания интерфейса
- кнопка `Connect` запускает синхронизацию и после её завершения продолжает вход на сервер
- кнопка `Download` выполняет только синхронизацию и не пытается сразу подключить игрока

## Проверка синхронизации

- клиент получает манифест от сервера
- недостающие файлы скачиваются
- уже актуальные файлы не скачиваются повторно без причины
- `public_http_base_url` работает, если он настроен
- прямой доступ по `server_http_port` работает, если `public_http_base_url` не задан

## Проверка cleanup

- файл, удалённый с сервера, удаляется у клиента только если он всё ещё совпадает с последней managed-версией
- локально изменённый managed-файл не удаляется автоматически
- защищённые файлы самого ModSync не попадают под cleanup
- после cleanup не остаются неожиданные пустые каталоги в управляемых путях

## Проверка UX

- экран прогресса открывается в ожидаемый момент
- логи синхронизации отображаются на экране прогресса
- после изменений, требующих перезапуска, показывается экран перезапуска
- при отсутствии изменений ModSync не показывает лишние предупреждения

## Проверка логов

- ошибки скачивания понятны по сообщению
- ошибки получения манифеста попадают в лог
- skip-события cleanup логируются отдельно от реального удаления

## Минимальный сценарий перед релизом

1. Запустить сервер с новой сборкой.
2. Подключиться клиентом без части нужных файлов.
3. Проверить успешную докачку и вход на сервер.
4. Удалить или заменить один managed-файл и проверить поведение cleanup.
5. Проверить лог и changelog перед финальной публикацией.
