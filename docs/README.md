## Скриншоты

Скриншоты в комплект пока не включены.

Причина:
- текущей текстовой документации достаточно для установки и поддержки
- для качественных скриншотов нужен отдельный воспроизводимый GUI-сеанс с заранее подготовленными состояниями интерфейса

Если понадобится, отдельно можно подготовить пакет скриншотов для:
- экрана списка серверов
- экрана прогресса загрузки
- экрана перезапуска
- примера ошибки входа без ModSync

## Правило обновлений

После каждого изменения в проекте нужно:
- поднять версию мода
- записать изменения в `CHANGELOG.md`
- обновить документацию, если изменение влияет на установку, использование или поддержку

В `1.0.89` дополнительно усилена сетевая устойчивость синхронизации:
- локальные/integrated-сессии больше не получают ложный кик по отсутствующему ModSync handshake
- таймауты загрузки манифеста и файлов увеличены для больших и медленных сборок
- если `public_http_base_url` в манифесте указывает на недоступный адрес, клиент теперь пробует скачать тот же файл с адреса выбранного сервера и скрытого HTTP-порта из MOTD
- после успешного pre-join sync клиент на dedicated-сервере теперь всё равно отправляет server-side подтверждение handshake, чтобы не получать кик через 30 секунд после входа
- `modsync.jar` теперь тоже может обновляться как управляемый мод: новая версия кладётся в staging, а после полного закрытия Minecraft внешний updater переносит её в `mods/`
- серверный handshake tracker теперь корректно переживает гонку между ранним клиентским `hello` и поздней регистрацией pending state, поэтому игрок не должен получать плавающий таймаут уже после входа в мир

В `1.0.91` публичное имя проекта переименовано в `SyncBridge`.
Технический `modsync` mod id сохранён, чтобы не ломать совместимость существующих установок, jar-имени и сетевой синхронизации.

В `1.0.92` timeout handshake на dedicated сервере увеличен до 120 секунд.
По умолчанию мод больше не кикает игрока автоматически сразу после handshake timeout: теперь пишет подробный лог в консоль, уведомляет админов и ждёт явного решения `allow` или `kick`.
